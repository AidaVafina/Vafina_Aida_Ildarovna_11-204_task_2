# Токенизация и лемматизация документов (задание 2)

Скрипт решает задание:
1. Извлекает токены из сохранённых документов.
2. Удаляет дубликаты, числовые токены, союзы, предлоги и мусор.
3. Группирует токены по леммам.

## Быстрый старт

```bash
python3 tokenize_and_lemmatize.py
```

По умолчанию скрипт берёт документы из папки `documents/` и сохраняет результаты в файлы:
- `tokens.txt`
- `lemmatized_tokens.txt`

Если документы лежат в других местах, укажите пути явно:

```bash
python3 tokenize_and_lemmatize.py /path/to/doc1.pdf /path/to/doc2.txt /path/to/folder
```

## Формат выходных файлов

1. `tokens.txt`:
   - каждая строка — один токен (`<токен>\n`)

2. `lemmatized_tokens.txt`:
   - каждая строка — лемма и список токенов (`<лемма> <токен1> <токен2> ... <токенN>\n`)

## Что фильтруется

- дубликаты токенов;
- числа и смешанные буквенно-цифровые фрагменты;
- HTML-разметка и служебный мусор;
- короткие и стоп-слова;
- союзы, предлоги и числительные (при доступном морфоанализаторе).

## Зависимости

- Python 3.10+
- Для точной лемматизации желательно наличие `pymorphy3` (или `pymorphy2`).
- Для PDF подходит `pdftotext` (poppler) или `pdfminer.six`.

См. `DEPLOYMENT_MANUAL.md` для подробных инструкций установки.

## Структура проекта

- `tokenize_and_lemmatize.py` — основной скрипт
- `tokens.txt` — список токенов (результат)
- `lemmatized_tokens.txt` — леммы и связанные токены (результат)
- `documents/` — папка для входных файлов (необязательно)
- `DEPLOYMENT_MANUAL.md` — инструкция по развёртыванию
- `RELEASE_NOTES.md` — заметки о версии
